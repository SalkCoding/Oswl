아래 설명은 기술적 배경 없이도 이해할 수 있도록 쉽게 풀어 작성했습니다.

### 기능 1: "이번 업데이트에서 뭐가 바뀌었나요?" — 버전 변경점 비교

**우리 서비스가 하는 일을 먼저 이해해 볼게요.**

OsWL은 소프트웨어 프로젝트가 사용하고 있는 외부 라이브러리(다른 개발자가 만들어 놓은 코드 부품)들을 자동으로 분석해서, 그 안에 보안 취약점이나 법적 문제(라이선스 위반)가 있는지 알려주는 서비스예요.

**그런데 지금은 어떤 문제가 있냐면...**

개발팀이 소프트웨어를 업데이트할 때마다 ("버전 1.0" → "버전 1.1") 새로 분석 결과를 올리게 됩니다. 그런데 지금은 새 분석 결과가 들어오면 그냥 최신 결과만 보여줄 뿐, **"이번 업데이트에서 뭐가 달라졌는지"를 한눈에 볼 수 있는 방법이 없어요.**

예를 들어, 어제는 100개의 라이브러리를 쓰고 있었는데 오늘 업데이트하니 102개가 됐다면:
- 새로 2개가 추가됐나요?
- 기존 1개가 업데이트되고 1개가 삭제됐나요?
- 새로 추가된 라이브러리에 보안 문제가 있나요?

이걸 직접 두 화면을 왔다 갔다 하며 비교해야 하는 상황이에요.

**우리가 추가하려는 것:**

소프트웨어 버전이 바뀌었을 때 **자동으로 변경된 내용을 요약해서 보여주는 페이지**를 만들려고 합니다. 마치 "이번 달 건강검진 결과와 지난달 결과를 비교해서 어디가 좋아지고 나빠졌는지 알려주는 리포트"처럼요.

### 기능 2: "실수로 삭제했어요!" — 30일 휴지통 복구

**지금은 어떻게 되냐면...**

현재 프로젝트 삭제 버튼을 누르면 그 즉시, 영구적으로 데이터가 사라집니다. 6개월치 분석 데이터가 클릭 한 번에 완전히 없어질 수 있어요.

**이 기능이 왜 필요하냐면...**

우리가 쓰는 이메일, 파일 앱, 메모 앱 등 대부분의 서비스는 "삭제" 버튼을 눌러도 바로 없어지지 않고 **"휴지통"에 30일 동안 보관**됩니다. 그 안에 실수를 발견하면 되살릴 수 있죠.

OsWL도 마찬가지로, 분석 데이터는 팀이 오랫동안 공들여 쌓아온 중요한 자산입니다. 실수로 또는 오해로 삭제 버튼을 눌렀을 때 복구할 수 있는 안전망이 필요합니다.

**우리가 추가하려는 것:**

- 삭제 버튼을 누르면 즉시 삭제되는 게 아니라 **"보관함(휴지통)"으로 이동**합니다.
- 프로젝트 목록 화면 어딘가에 **"삭제된 프로젝트 보기"** 영역을 만들어, 그곳에서 복구하거나 완전 삭제를 선택할 수 있게 합니다.
- 30일이 지나면 자동으로 영구 삭제됩니다. (사용자에게는 "○○일 후 완전 삭제 예정"으로 남은 기간을 보여줍니다.)

### 기능 3: "이 버전 분석 결과는 이제 필요 없어요" — 특정 버전 삭제

**지금은 어떻게 되냐면...**

지금은 프로젝트 전체를 삭제하는 기능만 있습니다. 예를 들어 버전 1.0, 1.1, 1.2, 1.3까지 분석 결과가 쌓여있을 때, 오래된 버전 1.0의 데이터만 깔끔하게 정리하고 싶어도 방법이 없습니다.

**왜 이게 문제냐면...**

- 오래된 분석 데이터가 쌓이면 화면이 복잡해지고 관리가 어려워집니다.
- 특히 매주 새 버전이 배포되는 팀이라면 수십 개의 버전이 쌓일 수 있어요.
- 전체 삭제 외에 선택적으로 정리할 수단이 없으면 불필요한 데이터가 계속 늘어납니다.

**우리가 추가하려는 것:**

프로젝트 내부 화면에서 버전을 선택하는 드롭다운 메뉴 근처에 **"이 버전 삭제" 옵션**을 추가합니다. 단, 이 버전 삭제도 소프트 삭제(휴지통 30일 보관)와 동일한 방식으로 동작합니다.

## 3. 기능별 화면 설계 (ASCII Proto UI)

> **표기 방법 안내:**
> - `[ ]` : 버튼
> - `★` : **신규 추가 요소** (디자인 작업 필요)
> - `···` : 기존 콘텐츠 (변경 없음, 레이아웃 유지)
> - 각 다이어그램은 **실제 현재 화면 구조**를 기반으로 그려졌으며, `★` 표시된 영역만 신규 반영됩니다.

#### 1-B. 버전 드롭다운 — "버전 비교하기" 메뉴 항목 추가

현재 버전 드롭다운은 다크 배경의 플로팅 패널에 버전 목록만 나열되어 있습니다.

### 기능 2: 소프트 삭제 및 휴지통 복구

## 4. 디자인 체크리스트

디자인 작업 시 아래 항목을 고려해 주세요.

### 공통
- [ ] 기존 디자인 언어(Tailwind 기반 색상, 타이포그래피, 컴포넌트)와 일관성 유지
- [ ] 모든 모달은 ESC 키 및 외부 클릭으로 닫힘
- [ ] 파괴적 액션(삭제, 영구삭제)은 빨간 계열 색상으로 강조
- [ ] 복구/안전한 액션은 파란 또는 초록 계열로 구분

### 기능 1: 버전 비교
- [ ] 변경 유형(추가/제거/버전업)별 시각적 색상 코딩 (예: 초록/빨강/파랑)
- [ ] 버전이 하나뿐인 경우, 비교 배너 및 버튼 숨김 처리
- [ ] 비교 결과가 없을 때(변경 없음) 빈 상태(empty state) 디자인 필요

### 기능 2: 소프트 삭제
- [ ] 남은 기간 D-day 배지: 10일 미만 🔴, 20일 미만 🟡, 20일 이상 🟢
- [ ] 휴지통 탭의 배지 숫자는 실시간 반영
- [ ] 휴지통이 비어있을 때 빈 상태 디자인 필요 ("삭제된 프로젝트가 없습니다")

### 기능 3: 버전 삭제
- [ ] 버전이 1개뿐일 때 삭제 버튼 비활성화 + 툴팁 표시 ("마지막 버전은 삭제할 수 없습니다")
- [ ] 삭제된 버전은 드롭다운에서 숨김 (휴지통 탭에서는 확인 가능)
