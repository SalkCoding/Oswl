# 버전 비교

버전 비교 페이지를 사용하면 두 스캔 결과를 나란히 비교하여 버전 간에 어떤 컴포넌트가 추가, 제거 또는 변경되었는지 확인할 수 있습니다.

URL: `/projects/{id}/version-diff`

---

## 사용 방법

1. 프로젝트를 열고 사이드바에서 **버전 비교**로 이동합니다.
2. 왼쪽 드롭다운에서 **기준(Base)** 스캔(이전 버전)을 선택합니다.
3. 오른쪽 드롭다운에서 **비교(Compare)** 스캔(새 버전)을 선택합니다.
4. **비교**를 클릭합니다.

OsWL이 비교를 계산하고 세 개의 섹션으로 표시합니다:

---

## 비교 섹션

### 추가된 컴포넌트

**비교** 스캔에는 있지만 기준 스캔에는 **없는** 컴포넌트.

두 버전 사이에 추가된 새 의존성입니다. CRITICAL 또는 HIGH CVE가 있는 항목은 즉시 주의하세요 — 새로운 공격 표면입니다.

### 제거된 컴포넌트

**기준** 스캔에는 있지만 비교 스캔에는 **없는** 컴포넌트.

제거된 의존성입니다. 알려진 CVE가 있는 항목의 제거는 긍정적인 신호입니다.

### 변경된 컴포넌트

**두 스캔 모두**에 있지만 **다른 버전**인 컴포넌트.

| 컬럼 | 설명 |
|---|---|
| **컴포넌트** | 라이브러리 이름 |
| **기준 버전** | 기준 스캔의 버전 |
| **비교 버전** | 비교 스캔의 버전 |
| **CVE 변화량** | CVE 수의 변화 (예: −2 CRITICAL은 두 개의 CRITICAL CVE가 해결됨을 의미) |

CVE를 해결하는 업그레이드는 음수 CVE 변화량(녹색)을 보입니다. 다운그레이드 또는 새 CVE를 도입하는 버전 변경은 양수 변화량(빨강)을 보입니다.

---

## 활용 사례

* **릴리즈 전 검토**: 릴리즈 후보를 마지막 릴리즈와 비교하여 새로운 고심각도 CVE가 도입되지 않았는지 확인.
* **의존성 감사**: 두 기능 브랜치를 비교하여 PR의 의존성 변경에 따른 리스크 영향 파악.
* **수정 검증**: 특정 라이브러리 업그레이드가 예상된 CVE를 실제로 해결했는지 확인.
* **회귀 감지**: 의존성 해결 충돌로 인한 의도치 않은 버전 다운그레이드 식별.

---

## 팁

* 비교는 저장된 스캔 데이터에서 요청 시 계산됩니다 — 재스캔이 필요 없습니다.
* 두 스캔의 컴포넌트 목록이 동일하고 버전 변경이 없으면, 세 섹션 모두 비어 있습니다(동일한 버전).
* 비교할 스캔 ID를 찾으려면 [스캔 히스토리](Scan-History.md)를 사용하세요.
