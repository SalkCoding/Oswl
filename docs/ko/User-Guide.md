# 사용자 가이드

이 페이지는 OsWL 웹 대시보드의 일상적인 사용법을 설명합니다.

**시스템 관리자**가 아니면 **멤버로 등록된 프로젝트**만 보입니다. 프로젝트 안에서 할 수 있는 일은 **역할 템플릿**(Admin / Developer / Viewer)에 따릅니다. [권한 레이어](Authorization-Layers.md) 참고.

---

## 프로젝트 대시보드 (`/projects`)

프로젝트 대시보드는 로그인 후 첫 화면입니다. 모든 활성 프로젝트를 카드 형식으로 보여줍니다.

### 프로젝트 카드

각 카드에 표시되는 정보:

| 필드 | 설명 |
|---|---|
| **이름** | 프로젝트 표시 이름 |
| **버전** | 최신 스캔의 버전 문자열 |
| **VCS 배지** | VCS에서 임포트된 경우 GitHub / GitLab / Bitbucket 아이콘 |
| **브랜치** | 가장 최근에 임포트된 브랜치 |
| **마지막 스캔** | 가장 최근 완료된 스캔의 상대적 타임스탬프 |
| **리스크 배지** | CRITICAL / HIGH / MEDIUM / LOW CVE 수 |
| **라이선스 배지** | 가장 심각한 라이선스 상태 (Restricted / Caution / Permitted / Unknown) |

### 프로젝트 액션

프로젝트 카드를 클릭하면 **보안 센터**가 열립니다.  
카드의 더보기 메뉴(⋮)에서 다음 작업을 수행할 수 있습니다:

* **상세 보기** — 보안 센터
* **라이선스 분석** — 라이선스 리스크 페이지
* **리스크 트렌드** — 히스토리 차트
* **스캔 히스토리** — 모든 스캔 목록
* **휴지통으로 이동** — 프로젝트 소프트 삭제

---

## 프로젝트 생성

프로젝트를 등록하는 두 가지 방법:

1. **Quick Import** — VCS 계정을 연결하고 저장소/브랜치를 선택합니다. [Quick Import](Quick-Import.md) 참조.
2. **CLI 푸시** — API 키로 프로젝트를 생성한 후 빌드 파이프라인에서 스캔 페이로드를 제출합니다. [CLI 연동](CLI-Integration.md) 참조.

---

## 휴지통

삭제된 프로젝트는 **휴지통** 섹션(프로젝트 목록 하단)에 표시됩니다.

| 작업 | 설명 |
|---|---|
| **복원** | 프로젝트를 활성 상태로 되돌림 |
| **영구 삭제** | 모든 데이터(스캔, CVE 등) 복구 불가 삭제 |
| **선택 복원** | 여러 프로젝트 일괄 복원 |
| **휴지통 비우기** | 휴지통의 모든 프로젝트 영구 삭제 |

---

## 프로젝트 내 탐색

프로젝트 내부에서 사이드바를 통해 다음 섹션에 접근할 수 있습니다:

| 섹션 | URL 패턴 | 설명 |
|---|---|---|
| 보안 센터 | `/projects/{id}/security-center` | CVE 목록 및 상태 관리 |
| 라이선스 분석 | `/projects/{id}/license` | 의존성 라이선스 컴플라이언스 |
| 리스크 트렌드 | `/projects/{id}/risk-trend` | 히스토리 리스크 차트 |
| 버전 비교 | `/projects/{id}/version-diff` | 두 스캔 비교 |
| 스캔 히스토리 | `/projects/{id}/scan-history` | 모든 과거 스캔 |
| CLI / API 키 | 설정 → CLI 탭 | 프로젝트 API 키 관리 |

## 컴포넌트 상세

보안 센터 또는 라이선스 뷰에서 컴포넌트(라이브러리)를 클릭하면 **컴포넌트 상세** 패널이 열립니다.

표시 내용:

* 전체 이름, 버전, 에코시스템
* CVSS 점수, 설명, 수정 버전이 포함된 CVE 목록
* 라이선스 이름 및 컴플라이언스 상태
* AI 생성 라이선스 리스크 요약 (AI가 설정된 경우)
* 패치 가능 여부 (Patchable / Non-Patchable / Unknown)
* 최신 버전 및 지원 중단 공지 (deps.dev 기반)
* 의존성 경로 (직접 의존성 vs. 전이적 의존성)
