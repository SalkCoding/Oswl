# 스캔 히스토리

스캔 히스토리 페이지는 프로젝트에 제출된 모든 스캔을 역순으로 나열합니다.

URL: `/projects/{id}/scan-history`

---

## 표시 내용

| 컬럼 | 설명 |
|---|---|
| **스캔 ID** | 내부 숫자 식별자 |
| **버전** | 스캔 시점의 프로젝트 버전 문자열 |
| **상태** | PENDING / SCANNING / ANALYZING / COMPLETED / FAILED |
| **컴포넌트** | 감지된 총 컴포넌트 수 |
| **제출 일시** | 스캔 제출 타임스탬프 |
| **제출자** | 사용자 이메일 (인증된 CLI 스캔 또는 Quick Import인 경우) |

---

## 스캔 상태

| 상태 | 설명 |
|---|---|
| `PENDING` | 스캔 수신됨; 처리 대기 중 |
| `SCANNING` | 의존성 매니페스트 파싱 중 |
| `ANALYZING` | CVE 및 라이선스 보강 진행 중 (NVD / OSV / deps.dev) |
| `COMPLETED` | 모든 보강 완료; 보안 센터 최신 상태 |
| `FAILED` | 오류 발생; 스캔 행의 오류 메시지 확인 |

---

## 스캔 삭제

스캔 행의 **삭제** 아이콘을 클릭하면 해당 스캔 레코드와 모든 컴포넌트 데이터가 영구적으로 제거됩니다.

> ⚠️ 이 작업은 되돌릴 수 없습니다. 스캔을 삭제하면 리스크 트렌드 차트에서도 제거됩니다.

---

## 스캔 재제출

OsWL은 **동일한 버전** 문자열에 대한 스캔 재제출을 지원합니다. 서버가 기존 스캔 레코드를 초기화하고 새 페이로드를 재처리합니다. 이전 스캔이 보강 중간에 실패했거나, 버전 변경 없이 의존성이 변경된 경우에 유용합니다.

재스캔 방법: CLI를 통해 동일한 버전 제출 (`POST /api/scan`) — OsWL이 버전 일치를 감지하고 자동으로 레코드를 초기화합니다.
