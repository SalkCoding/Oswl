# 보안 센터

보안 센터는 OsWL의 기본 취약점 관리 인터페이스입니다. 프로젝트의 최신 스캔에서 감지된 컴포넌트에 영향을 미치는 모든 CVE를 필터링·정렬 가능한 목록으로 제공합니다.

URL: `/projects/{id}/security-center`

**접근:** `SECURITY_CENTER_VIEW`(또는 시스템 관리자) 및 [프로젝트 멤버십](Authorization-Layers.md) 필요. 보내기·인쇄는 `SECURITY_CENTER_EXPORT`, 상태 변경은 `SECURITY_CENTER_UPDATE_STATUS`.

---

## CVE 목록 이해하기

보안 센터의 각 행은 스캔된 컴포넌트와 연결된 하나의 **CVE**(Common Vulnerabilities and Exposures) 항목을 나타냅니다.

| 컬럼 | 설명 |
|---|---|
| **컴포넌트** | 라이브러리 이름과 버전 |
| **에코시스템** | 패키지 에코시스템 (MAVEN, NPM, PYPI, …) |
| **CVE ID** | deps.dev/OSV의 CVE 또는 GHSA 식별자 (예: `CVE-2021-44228`) |
| **CVSS 점수** | 수치 심각도 점수 (0.0–10.0) |
| **심각도** | CRITICAL / HIGH / MEDIUM / LOW |
| **수정 버전** | 권장 수정 버전 (알려진 경우) |
| **상태** | 현재 분류 상태 |
| **패치 가능성** | 수정 사항 존재 여부 |

---

## 심각도 수준

OsWL은 표준 CVSS 3.x 구간에 따라 기본 점수를 심각도로 분류합니다:

| 심각도 | CVSS 점수 범위 | 의미 |
|---|---|---|
| **CRITICAL** | 9.0 – 10.0 | 원격 악용 가능, 높은 영향도 — 즉시 수정 필요 |
| **HIGH** | 7.0 – 8.9 | 심각한 위험 — 이번 스프린트 내 우선 처리 |
| **MEDIUM** | 4.0 – 6.9 | 보통 위험 — 가까운 시일 내 수정 |
| **LOW** | 0.1 – 3.9 | 최소 위험 — 편의에 따라 수정 |
| **NONE** | 0.0 | 정보 제공용 / 활성 점수 없음 |

## CVE 상태 라이프사이클

프로젝트의 모든 CVE에 다음 상태 중 하나를 지정할 수 있습니다:

| 상태 | 의미 |
|---|---|
| **OPEN** | 새로 감지됨 — 검토 필요 |
| **IN_PROGRESS** | 적극적으로 수정 중 |
| **SUPPRESSED** | 인지된 위험; 현재 맥락에서 즉각적 우선순위 아님 |
| **FALSE_POSITIVE** | 적용 불가로 확인됨 (예: 취약한 코드 경로가 도달 불가) |
| **RESOLVED** | 수정 완료 — 업그레이드 적용 또는 컴포넌트 제거됨 |

> 상태 업데이트에 필요한 권한: `SECURITY_CENTER_UPDATE_STATUS` 또는 시스템 관리자.

---

## 필터링 및 정렬

보안 센터 상단의 필터 바를 사용하여 목록을 좁힐 수 있습니다:

* **심각도** — CRITICAL / HIGH / MEDIUM / LOW
* **상태** — OPEN / IN_PROGRESS / SUPPRESSED / FALSE_POSITIVE / RESOLVED
* **에코시스템** — MAVEN / NPM / PYPI / 기타
* **패치 가능성** — Patchable / Non-Patchable / Unknown
* **검색** — CVE ID 및 컴포넌트 이름 전체 텍스트 검색

컬럼 헤더를 클릭하면 오름차순/내림차순으로 정렬됩니다.

---

## 일괄 상태 업데이트

1. 체크박스를 사용하여 하나 이상의 CVE를 선택합니다.
2. 작업 도구 모음에서 **상태 업데이트**를 클릭합니다.
3. 드롭다운에서 새 상태를 선택합니다.
4. 확인합니다.

선택된 모든 CVE가 단일 트랜잭션으로 업데이트됩니다.

---

## 컴포넌트 상세

컴포넌트 이름을 클릭하면 **컴포넌트 상세** 사이드 패널이 열립니다:

* 해당 라이브러리의 모든 CVE (전체 CVSS 분석 포함)
* 라이선스 이름 및 컴플라이언스 상태
* AI 생성 라이선스 리스크 요약
* 최신 버전 및 지원 중단 공지
* 전체 의존성 경로 (직접 vs. 전이적)

## AI 보안 인사이트

AI 제공업체가 설정된 경우(**설정 → AI**), 각 완료된 스캔은 보안 센터 상단에 AI 단락 요약을 생성합니다:

* **보안 현황 인사이트** — 현재 CVE 수 및 심각도 분포에 대한 전반적 평가
* **보안 리스크 트렌드 인사이트** — 이전 스캔과 비교하여 개선 또는 악화 여부 표시

AI 인사이트는 스캔당 한 번 생성되며, 새 스캔이 제출되지 않는 한 재생성되지 않습니다.

---

## 패치 가능성

OsWL은 라이브러리의 모든 CVE에서 `fixVersion` 필드를 기반으로 패치 가능성을 도출합니다:

| 상태 | 조건 |
|---|---|
| **PATCHABLE** | 알려진 수정 버전이 있는 CVE가 하나 이상 존재 |
| **NON_PATCHABLE** | CVE는 존재하지만 알려진 수정 사항 없음 |
| **UNKNOWN** | CVE 없음 또는 CVE 데이터 미보강 |

---

## 데이터 출처

CVE 데이터는 두 출처에서 가져와 병합됩니다:

* **deps.dev** — `GetAdvisory`로 GHSA 어드바이저리, CVE 별칭, CVSS 점수, 제목 조회
* **OSV** (Open Source Vulnerabilities) — `POST https://api.osv.dev/v1/querybatch`로 요약, 수정 버전, **CWE ID**(`database_specific.cwe_ids`) 조회

CWE 식별자(예: `CWE-79`)는 OSV가 제공할 때 컴포넌트 상세 패널에 표시됩니다.

보강은 각 스캔 후 자동으로 실행되며, **설정 → 캐시**(`/api/settings/cache`) 정책에 따라 갱신됩니다.
