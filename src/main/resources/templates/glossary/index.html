Definition Dictionary

OsWL 시스템 내 모든 용어의 공식 정의 사전입니다. 디자이너·개발자·운영자가 동일한 언어로 소통하기 위한 단일 출처(single source of truth)입니다.

Risk Level (Severity)

보안 취약점(CVE)의 심각도 수준입니다. Security Center 필터·컴포넌트 배지·Risk Trend 그래프에 아래 5가지 레이블로 표시됩니다. CVSS 3.x Score를 기준으로 자동 분류됩니다. 각 레이블 아래 회색으로 표시된 값은 백엔드 코드의 내부 enum 값입니다.

UI 레이블CVSS RangeDefinition
Critical
CRITICAL
9.0 – 10.0 즉각적인 조치가 필요한 최고 심각도 취약점. 원격 코드 실행(RCE), 인증 우회 등 시스템 전체를 위협하는 공격 벡터를 포함합니다.
High
HIGH
7.0 – 8.9 높은 심각도 취약점. 공격 복잡도가 낮거나 권한 상승이 가능한 경우. 빠른 패치 또는 임시 완화 조치가 권장됩니다.
Medium
MEDIUM
4.0 – 6.9 중간 심각도 취약점. 공격 조건이 제한적이거나 영향 범위가 한정적입니다. 다음 정기 릴리스 주기 내 패치를 권장합니다.
Low
LOW
0.1 – 3.9 낮은 심각도 취약점. 공격 성공 가능성이 매우 낮거나 영향이 미미합니다. 위험 수용 정책에 따라 대응 시점을 결정합니다.
None / Unscored
0.0 CVSS 점수가 없거나(Unscored) 취약점이 없는 상태입니다. 해당 라이브러리에 등록된 CVE가 없거나, CVE는 있지만 아직 CVSS 점수가 부여되지 않은 경우 이 레이블이 표시됩니다.

Patchability

취약점을 패치할 수 있는지 여부를 나타냅니다. CVE에 fixVersion이 존재하는지를 기준으로 자동 계산됩니다. 취약점이 없는 컴포넌트에는 이 값 대신 Version Health 배지가 표시됩니다.

Patchable
하나 이상의 CVE에 fixVersion이 존재하여 버전을 업그레이드함으로써 취약점을 제거할 수 있는 상태입니다. 가장 높은 심각도 CVE의 fixVersion을 "Best Fix Version"으로 표시합니다.
Non-Patchable
CVE는 존재하지만 알려진 수정 버전이 없는 상태입니다. 라이브러리 교체, 격리, 또는 임시 완화 조치가 필요합니다.
Unknown
CVE 데이터가 아직 분석되지 않았거나 CVSS 정보가 없어 패치 가능 여부를 판단할 수 없는 상태입니다.

Version Health

취약점(CVE)이 없는 컴포넌트에 표시되는 버전 상태입니다. deps.dev 메타데이터를 기반으로 자동 계산되며, Patchability 배지와 동시에 표시되지 않습니다 (CVE 유무로 둘 중 하나만 표시).

Up-to-date deps.dev 기준으로 현재 사용 중인 버전이 해당 패키지의 최신 안정(stable) 버전과 일치합니다. 별도 조치가 필요 없습니다.
Outdated 현재 버전보다 최신 안정 버전이 존재합니다. CVE는 없지만 새로운 기능·버그 수정이 포함된 버전으로 업그레이드를 권장합니다. UI에서 "→ v{latestVersion}" 형식으로 권장 버전을 함께 표시합니다.
Deprecated 패키지 관리자(deps.dev) 또는 유지보수 팀이 공식적으로 사용 중단을 선언한 버전 또는 패키지입니다. 더 이상 보안 패치나 업데이트가 제공되지 않으므로 대체 패키지로의 마이그레이션이 필요합니다. deps.dev가 제공하는 사용 중단 사유 메시지를 함께 표시합니다.

License Status

각 라이브러리의 오픈소스 라이선스가 프로젝트의 라이선스 정책에 부합하는지 나타냅니다. SPDX 식별자를 기준으로 내장 정책 테이블과 매핑하여 자동 분류되며, Settings에서 커스터마이즈할 수 있습니다. License Status는 4단계 체계로, Security Severity(5단계)와 독립된 용어를 사용합니다. 각 레이블 아래 회색으로 표시된 값은 백엔드 코드의 내부 enum 값입니다.

Restricted
RESTRICTED
상업적 제품에 사용 시 법적 위험이 높은 카피레프트(copyleft) 또는 비상업적 라이선스입니다. 즉각적인 법무 검토 및 대체 라이브러리 사용이 필요합니다. 기본 예시: GPL-2.0, GPL-3.0, AGPL-3.0, SSPL-1.0, BUSL-1.1, CC-BY-NC-4.0.
Caution
CAUTION
조건부 허용 라이선스입니다. 사용 가능하지만 소스 공개 의무, 특허 조항 등의 조건이 붙어 법무 검토 후 사용을 권장합니다. 기본 예시: LGPL-2.1, LGPL-3.0, MPL-2.0, EPL-2.0, EUPL-1.2.
Permitted
PERMITTED
상업적 사용 및 재배포에 제한이 없는 허용적(permissive) 라이선스입니다. 별도 법무 검토 없이 사용 가능합니다. 기본 예시: MIT, Apache-2.0, BSD-2-Clause, BSD-3-Clause, ISC, Unlicense, 0BSD, CC0-1.0.
Unknown
UNKNOWN
deps.dev에서 SPDX 식별자를 반환하지 않았거나 시스템 정책 테이블에 등록되지 않은 라이선스입니다. 수동 검토가 필요합니다.

SPDX

Software Package Data Exchange. 오픈소스 라이선스 정보를 표준화하기 위한 식별자 체계입니다. 예: MIT, Apache-2.0, GPL-3.0-only. OsWL은 deps.dev가 반환하는 SPDX 식별자를 라이선스 정책 매핑의 키로 사용합니다.

Scan Status

CLI가 전송한 스캔의 처리 단계입니다. Scan History 화면에서 확인할 수 있습니다.

PENDING CLI가 스캔 데이터를 전송 중인 상태입니다. 서버가 패키지 목록을 수신하고 있습니다.
SCANNING 패키지 목록 저장 완료. 취약점 분석 대기 중입니다.
ANALYZING deps.dev 및 OSV API에서 CVE·라이선스 데이터를 수집하는 중입니다. NVD API Key가 설정된 경우 추가적인 CVSS 점수 강화(enrichment)도 이 단계에서 수행됩니다.
COMPLETED 분석 완료. Security Center 및 License 탭에서 결과를 확인할 수 있습니다.
FAILED 외부 API 오류, 네트워크 타임아웃 등으로 분석에 실패했습니다. Scan History에서 오류 메시지를 확인하고 다시 스캔을 실행하세요.

Import Source

프로젝트 버전이 OsWL에 등록된 방법을 나타냅니다.

CLI OsWL CLI 도구를 로컬 머신 또는 빌드 서버에서 직접 실행하여 스캔 데이터를 전송한 경우입니다. 소스코드 업로드 없이 패키지 메타데이터만 전송됩니다.
GIT GitHub Integration 패널을 통해 GitHub 저장소를 연결하고 브랜치를 선택하여 가져온 경우입니다. OAuth 인증 후 저장소의 의존성 파일을 자동으로 분석합니다.

Dependency Terms

Direct Dependency 프로젝트가 직접 선언한 의존성(depth 1). 빌드 파일(pom.xml, package.json 등)에 명시적으로 기재된 패키지입니다. 컴포넌트 상세 패널에서 "Direct" 배지로 표시됩니다.
Transitive Dependency 직접 의존성이 다시 의존하는 간접 의존성(depth 2+). 프로젝트 빌드 파일에는 없지만 실행 시 함께 포함됩니다. 컴포넌트 상세 패널에서 "Via N intermediate(s)" 형식으로 경로를 표시합니다.
Dependency Path 루트 프로젝트에서 특정 라이브러리까지의 전체 의존성 경로입니다. 하나의 라이브러리가 여러 경로를 통해 포함될 수 있으며, 각 경로가 별도로 표시됩니다. 예: my-app → spring-web → commons-lang3
Depth 의존성 경로의 노드(패키지) 수입니다. depth 2 = Direct(루트 → 라이브러리), depth 3 이상 = Transitive(루트 → 중간 패키지 → 라이브러리).
Dependency Info Security Center 테이블의 컴포넌트 행에 표시되는 요약 문자열입니다. 예: Direct (2) + Transitive (5). CLI가 스캔 시 계산하여 전송합니다.
Ecosystem 패키지가 속한 언어/런타임 생태계입니다. deps.dev의 시스템 파라미터와 동일하게 사용됩니다. 지원 값: MAVEN · NPM · PYPI · GO · CARGO · NUGET · RUBYGEMS.

Component States

Security Center에서 각 컴포넌트에 부여할 수 있는 검토 상태입니다. Bulk Action을 통해 여러 컴포넌트에 일괄 적용할 수 있습니다.

Reviewed 담당자가 해당 컴포넌트의 취약점을 인지하고 검토를 완료했음을 표시하는 상태입니다. 취약점이 제거된 것은 아니며, 팀 내 인식(awareness) 및 작업 추적 용도로 사용합니다.
Ignored 해당 컴포넌트의 취약점을 의도적으로 무시하겠다고 표시하는 상태입니다. 예를 들어 테스트 전용 의존성이거나 공격 표면에 해당하지 않는 경우에 사용합니다. Ignored 상태의 컴포넌트는 필터로 숨길 수 있습니다.

CVE / Advisory Terms

CVE Common Vulnerabilities and Exposures. 공개된 보안 취약점의 표준 식별자입니다. 예: CVE-2021-44228 (Log4Shell). NVD(National Vulnerability Database)에서 관리합니다.
GHSA GitHub Security Advisory. GitHub이 관리하는 보안 권고 식별자입니다. 예: GHSA-jfh8-c2jp-hdp8. deps.dev가 주로 GHSA ID를 반환하며, OsWL은 이를 CVE ID로 매핑합니다.
CVSS Score Common Vulnerability Scoring System. 취약점 심각도를 0.0~10.0 사이 점수로 수치화합니다. OsWL은 CVSS 3.x 기준을 사용합니다. 점수 출처는 우선순위: NVD(NVD API Key 설정 시) > deps.dev cvss3Score.
CVSS Vector CVSS 점수를 구성하는 세부 지표들의 문자열 표현입니다. 예: CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:C/C:H/I:H/A:H. 공격 벡터(AV), 공격 복잡도(AC), 필요 권한(PR) 등을 나타냅니다.
CWE Common Weakness Enumeration. 소프트웨어 취약점의 근본 원인(weakness) 유형 분류입니다. 예: CWE-79(XSS), CWE-89(SQL Injection). NVD API Key 설정 시에만 표시됩니다.
Fix Version 해당 CVE가 수정된 가장 초기의 라이브러리 버전입니다. 이 버전 이상으로 업그레이드하면 해당 취약점이 제거됩니다. Patchability 계산에 직접 사용되는 핵심 필드입니다.
Best Fix Version 컴포넌트에 연결된 CVE 중 가장 높은 심각도의 CVE가 가진 Fix Version입니다. 단 하나의 업그레이드 목표 버전으로 안내하기 위해 UI에 표시됩니다.

AI Terms

AI Summary 설정된 AI 모델(OpenAI·Gemini·Anthropic·Local)이 생성한 CVE 한 줄 요약입니다. 기술 문서를 읽기 어려운 팀원을 위해 취약점의 영향과 위험을 쉬운 언어로 설명합니다. AI가 설정되지 않으면 표시되지 않습니다.
AI Provider AI Summary 생성에 사용할 모델 공급사입니다. 지원 값: OPENAI(GPT) · GEMINI · ANTHROPIC(Claude) · LOCAL(Ollama 등 OpenAI-compatible 엔드포인트). Settings › AI 탭에서 설정합니다.

External Data Sources

deps.dev Google이 운영하는 오픈소스 패키지 메타데이터 서비스입니다. OsWL은 deps.dev API를 통해 라이선스 정보, 최신 버전, deprecated 여부, GHSA advisory 목록을 수집합니다.
OSV Open Source Vulnerabilities. Google이 운영하는 오픈소스 취약점 데이터베이스입니다. OsWL은 OSV querybatch API를 통해 취약점 상세 정보(summary, aliases)를 보완합니다.
NVD National Vulnerability Database. NIST가 운영하는 미국 국가 취약점 데이터베이스입니다. API Key 설정 시 CVSS Score·Vector와 CWE ID를 추가로 수집합니다(NVD enrichment). Settings › External API 탭에서 설정합니다.

System / Project Terms

Project OsWL에서 분석하는 소프트웨어 단위입니다. GitHub의 owner/repo 또는 CLI로 등록된 독립 프로젝트에 대응합니다. 하나의 Project 아래 여러 브랜치(Project Version)가 관리됩니다.
Project Version 프로젝트의 브랜치 단위 임포트 기록입니다. 같은 브랜치를 다시 임포트하면 기존 row가 갱신되고, 새 브랜치를 임포트하면 새 row가 생성됩니다. Scan History 드롭다운에 버전 목록으로 표시됩니다.
API Key oswl_ 접두사가 붙은 SecureRandom 기반 토큰입니다. CLI 클라이언트가 스캔 데이터를 서버로 전송할 때 인증에 사용합니다. 프로젝트별로 발급되며, 만료 시각 설정 및 개별 취소(revoke)가 가능합니다.
Library (Shared) (name, version, ecosystem) 조합으로 식별되는 오픈소스 패키지의 글로벌 레코드입니다. CVE·라이선스 데이터는 Library 단위로 한 번만 저장되어 모든 프로젝트/스캔에서 공유됩니다. 동일 라이브러리를 여러 프로젝트가 사용해도 중복 fetch하지 않습니다.
Scan Component 스캔 결과(ScanResult)와 Library의 연결 엔티티입니다. 해당 스캔 맥락에서의 reviewed·ignored 상태와 의존성 경로(DependencyPath) 정보를 보유합니다. Security Center 테이블의 한 행이 이 단위에 대응합니다.
Version Diff 두 스캔 버전 사이에 추가(Added)·제거(Removed)·변경(Changed)된 컴포넌트 목록을 비교하는 기능입니다. 릴리스 간 보안 변화를 추적하는 데 사용합니다.
Permanent Cache Library 레코드를 영구적으로 캐시하는 정책입니다. 활성화 시 기존 Library 행을 deps.dev에서 다시 조회하지 않습니다. 비활성화 시 Cache TTL(일 단위)에 따라 오래된 레코드를 재조회합니다. Settings › Cache 탭에서 설정합니다.
System Admin OsWL 인스턴스의 최고 관리자 계정입니다. 시스템 최초 설정(Setup) 화면에서 1명만 생성됩니다. 모든 설정·사용자·프로젝트에 대한 전체 권한을 가집니다.
Trash Projects 목록에서 삭제된 프로젝트가 임시로 보관되는 공간입니다. 소프트 삭제(soft-delete) 방식으로 데이터는 유지되며 복원이 가능합니다. Trash에서 완전 삭제 시 데이터가 영구 제거됩니다.

이 사전은 코드베이스의 도메인 모델(Entity, Enum)과 UI 구현을 기반으로 자동 정리되었습니다. 새로운 용어가 추가되거나 정의가 변경될 경우 이 페이지를 함께 업데이트해 주세요.
