AI 기반 분석

AI를 활용하여 취약점의 실제 영향도를 분석하고, 우선순위와 패치 방향을 자동으로 제안합니다.

라이선스 관리

라이선스 충돌,
사전에 차단하세요

GPL·AGPL 등 카피레프트 라이선스가 포함된 경우 소스코드 공개 의무가 발생할 수 있습니다. OsWL은 라이선스 정책을 정의하고, 위반 여부를 자동으로 검사합니다.

커스텀 라이선스 정책 설정
팀의 기준에 맞게 허용·주의·위험 라이선스를 직접 지정하세요.
컴포넌트별 라이선스 확인
각 의존성의 라이선스 정보를 개별 조회하고 정책 준수 여부를 확인합니다.
라이선스 용어 사전 제공
복잡한 오픈소스 라이선스 조항을 쉽게 이해할 수 있는 용어 사전을 기본 제공합니다.
통합

기존 워크플로우에
자연스럽게 녹아듭니다

GitHub, CLI, REST API를 통해 기존 개발 파이프라인에 OsWL을 손쉽게 연동하세요.

GitHub 연동

GitHub 레포지토리를 직접 연결하여 브랜치별 의존성을 자동으로 임포트합니다. OAuth 및 Personal Access Token을 지원합니다.

CLI 클라이언트

CI/CD 파이프라인에 OsWL CLI를 통합하여 빌드 시 자동으로 스캔을 실행하고 결과를 업로드합니다.

REST API

표준 REST API로 모든 기능을 외부에서 제어합니다. API Key 인증 및 OpenAPI 문서를 기본으로 제공합니다.

CI/CD Pipeline — GitHub Actions
# OsWL SCA 스캔 실행
$ oswl-cli scan \
--project "my-backend-service" \
--sbom ./build/bom.json \
--api-key $OSWL_API_KEY

사용 방법

3단계로 시작하세요

복잡한 설정 없이 빠르게 프로젝트를 분석하고 보안 현황을 파악할 수 있습니다.

Step 01

프로젝트 임포트

GitHub 레포지토리를 연결하거나 CLI로 SBOM 파일을 업로드하여 프로젝트를 등록합니다.

Step 02

자동 분석

OsWL이 모든 의존성을 분석하고, CVE 데이터베이스와 매칭하여 취약점과 라이선스 리스크를 자동 탐지합니다.

Step 03

지속 모니터링

대시보드에서 위험도 추세를 추적하고, 새로운 CVE가 발견될 때마다 팀에게 즉시 알려줍니다.
